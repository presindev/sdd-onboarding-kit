Summary

Add a notes recent CLI command that prints the most recent notes, newest first, with an optional --limit N argument. This is the worked example spec shipped with the SDD onboarding kit; it shows what a finished, reviewed feature spec looks like.

Functional requirements

REQ-001 When the user runs notes recent without --limit, the system shall print at most five notes ordered by descending creation time.
REQ-002 When the user runs notes recent --limit N with N greater than zero, the system shall print at most N notes ordered by descending creation time.
REQ-003 If the user provides a non-positive or non-numeric limit, the system shall exit with a validation error and print no notes.

Non-functional requirements

NFR-001 The command shall complete in under one second for a store of 10,000 notes.

Edge cases

EDGE-001 When no notes exist, the system shall print the empty-state message No notes found. and exit successfully.

Error states

ERR-001 If the notes store cannot be read, the system shall print a clear error to stderr and exit with a non-zero status.

Acceptance criteria

AC-001 notes recent on a store with 8 notes prints exactly 5, newest first.
AC-002 notes recent --limit 2 prints exactly 2 notes, newest first.
AC-003 notes recent --limit 0 exits non-zero with a validation message and prints no notes.
