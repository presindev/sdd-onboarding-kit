Design — Recent notes CLI command

Task ID FEAT-007
Feature slug example-feature
Status done

Technical summary

Add a recent subcommand to the existing argparse-based CLI. Business logic lives in a new get_recent_notes(limit) function in src/notes.py, which sorts the in-memory note list by created_at descending and slices to limit. The CLI layer only parses and validates arguments; it holds no business logic.

Test design

One CLI-level test per requirement in tests/test_cli_recent.py, using the existing cli_runner fixture: default limit (REQ-001), custom limit (REQ-002), invalid limit (REQ-003), empty store (EDGE-001), unreadable store (ERR-001). The full matrix lives in acceptance-tests.html.

Risks and trade-offs

Sorting in memory is O(n log n) per call; acceptable for the 10k-note target (NFR-001). Rejected alternative: keeping a sorted index in the store — more code and a migration for no measurable gain at this scale.

Rollback plan

Revert the single feature commit. No data, schema or config changes exist, so removal is complete and safe.
